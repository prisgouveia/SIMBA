Sumário

Recentemente, a redução nos custos para a coleta de dados geográficos, aliada a fatores importantes como os avanços nas tecnologias de mapeamento digital e de comunicação, contribuiu para que a Internet se transformasse em um grande repositório de dados geográficos. Atualmente, dados geográficos são disponibilizados por diversos níveis de provedores (agências públicas de diversos níveis de governo, empresas privadas, instituições acadêmicas, pessoas comuns, etc) e acessados por vários tipos de clientes no mundo todo.

Apesar do grande volume de informações disponibilizadas, dados geográficos eram publicados de forma desorganizada e não padronizada, o que levava a grandes problemas de heterogeneidade. Tais problemas podiam ser observados através da existência de diferentes modelos de dados para representar as informações, o uso de diferentes plataformas para o acesso aos dados e aplicações de SIG (Sistemas de Informações Geográficas) com baixa interoperabilidade. Tudo isto tornava difícil as tarefas de acessar e integrar dados geográficos oferecidos por provedores diferentes, bem como compartilhar estes dados entre os muitos usuários e aplicações existentes na Internet. Ademais, esta dificuldade fazia com que muitas agências tivessem que gastar muito dinheiro e outros recursos na produção de dados que já tinham sido produzidos por outras organizações, e que poderiam ser reusados a custos bem mais baixos, ou, em alguns casos, sem nenhum custo.

Visando resolver os problemas gerados pela heterogeneidade e facilitar a interoperabilidade entre aplicações que usam dados espaciais e a integração de dados de fontes distintas, foram propostas recentemente as infraestruturas de dados espaciais (do inglês Spatial Data Infrastructures – SDI) [WILLIAMSON et al., 2003]. Estas infraestruturas podem ser definidas como uma base relevante de tecnologias, políticas e arranjos institucionais que facilitam a disponibilidade e acesso a dados espaciais, oferecendo uma base para descoberta, avaliação e aplicação para usuários e provedores de todos os níveis de governo, do setor comercial, do setor sem fins lucrativos, da academia e por cidadãos em geral [GSDI, 2004

Atualmente, infraestruturas de dados espaciais podem ser formadas por agências de governo de vários níveis (municipal, estadual, federal, internacional), por organizações pertencentes a uma mesma região geográfica, por comunidades que possuam interesses em comum ou até mesmo por cidadãos comuns. Desde a sua proposição, várias SDIs vem sendo desenvolvidas em todo o mundo. Exemplos importantes incluem a NSDI [FGDC, 2005], nos Estados Unidos, o Inspire [INSPIRE, 2008], na Europa, a ASDI [ANZLIC, 2003], na Oceania, e a GSDI [HOLLAND et al., 1999], uma iniciativa que propõe a criação de uma SDI de nível global. No Brasil, existe uma proposta para o desenvolvimento de uma SDI de nível nacional, chamada de INDE (Infraestrutura Nacional de Dados Espaciais) [CONCAR, 2010].

Um dos objetivos visados pelo desenvolvimento de uma SDI consiste em permitir que os dados geográficos já existentes possam ser facilmente localizados e acessados pelos seus clientes. Para atingir este objetivo, as infraestruturas atuais normalmente oferecem um serviço de catálogo [OGC, 2007], que é usado tanto por seus provedores de dados geográficos quanto por clientes. Os provedores de dados geográficos usam este serviço para anunciar os seus conjuntos de dados. Para isto, eles registram os seus conjuntos de dados no serviço. Durante este processo, eles fornecem uma série de informações acerca dos dados oferecidos, baseados no padrão de metadados adotado para a infraestrutura. Informações fornecidas durante este processo incluem o título do conjunto de dados, o nome da agência responsável pela publicação, a resolução dos dados, a extensão geográfica coberta pelos dados, a extensão temporal, um conjunto de palavras-chaves descritivas e a URL a partir da qual os dados podem ser acessados. Os clientes, por sua vez, utilizam o catálogo para localizar os dados geográficos de seu interesse.

Mesmo tendo facilitado a localização de dados geográficos, os serviços de catálogo oferecidos pelas SDI atuais ainda possuem grandes limitações.Algumas destas limitações surgem devido à forma como dados geográficos são oferecidos. Atualmente, por questões de padronização, dados geográficos são cada vez mais oferecidos através de serviços web padronizados, especificados pelo Open Geospatial Consortium (OGC). Exemplos importantes destes serviços incluem o Web Map Service (WMS) [OGC, 2004] e o Web Feature Service (WFS) [OGC, 2005].
